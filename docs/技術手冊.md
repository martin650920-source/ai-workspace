# ai-workspace — 技術手冊

---

## 一、整體架構概念

### 比喻

把 ai-workspace 想成一個「個人 AI 作業系統」：

```
你的電腦
└── Google Drive（備份 + 跨裝置同步）
    └── ai-workspace/（唯一本體）
            ↓ setup 腳本建立捷徑（symlink）
        AI 工具啟動時自動讀取
```

**核心設計原則：**
- **一份本體** — 所有設定只寫一次，多個 AI 工具共用
- **按需載入** — 只燒必要的 token，不用的不載
- **公私分離** — 骨架進 Git 公開，私密內容留本機
- **四層分工** — rules（該怎麼做）/ mem（發生過什麼）/ projects（事實是什麼）/ skills（怎麼執行），各自獨立演化

---

## 二、兩層載入架構

```
第一層：全域層（每次都載）
    rules/global.md       ← 你是誰、偏好、AI 行為準則（Rule/記憶/Skill 建立規則）

        +

第二層：專案層（自動偵測才載）
    projects/nagra-tntsat.md            ← 專案事實（/init 產生）
    rules/projects/nagra-tntsat/*.md    ← 專案規則（全域 + path-scoped）
    mem/projects/nagra-tntsat.md        ← 該專案累積筆記
```

**優先序（有衝突時）：專案層 > 全域層**

> 舊版的「情境層」（work/life + Secure Mode MCP 白名單）已於 v3 重構整套移除，日後有需要再重新設計。

## 三、Token 消耗說明

```
session 啟動
    ↓
CLAUDE.md（5行）           ← 固定消耗，極小
    ↓
SKILL.md（~150行）         ← 固定消耗，中型
    ↓
rules/global.md            ← 固定消耗，小
    ↓
projects/xxx.md            ← 偵測到才燒
rules/projects/xxx/*.md    ← 偵測到才燒
mem/projects/xxx.md        ← 偵測到才燒
    ↓
其他所有檔案               ← 0，永遠不自動載入
```

**設計紀律：**
- 規則只放「指令」，不放「內容」
- 內容放獨立 md，由 SKILL.md 按需讀取
- `rules/global.md` 建議控制在合理長度，內容量大到難維護時再拆分主題

## 四、MCP 架構

```
mcp/settings.json     ← 架構設定，只引用環境變數（${GITLAB_TOKEN} 等），進 Git
mcp/.env.example      ← 憑證欄位範本，進 Git
mcp/manifest.md        ← 各主機安裝狀態清單，進 Git（唯一允許自動 push 的例外）
~/.mcp.env             ← 真實憑證，使用者家目錄，不進 Git、不進 ai-workspace 目錄
```

新主機設定流程：複製 `.env.example` → `~/.mcp.env`，手動填入實際 key 值；shell 需
`source ~/.mcp.env` 才能讓 `${GITLAB_TOKEN}` 這類引用生效。

## 七、Session 啟動完整流程

```
1. Claude Code 啟動
        ↓
2. 讀 ~/.claude/CLAUDE.md
   → symlink 指向 adapters/claude/CLAUDE.md
   → 內容：「去讀 SKILL.md」
        ↓
3. 讀 skills/global/context-loader/SKILL.md
        ↓
4. 讀 rules/global.md（你是誰、規則、偏好）
        ↓
5. Skill Drift 偵測（skills/global/ 是否都連結到 ~/.claude/skills/）
        ↓
6. 掃 cwd 偵測 Marker Files
   有符合 → 確認，讀 projects/<name>.md +
            rules/projects/<name>/*.md（若有）+
            mem/projects/<name>.md（若有）
   沒符合 → 列清單讓你選，或跳過
        ↓
7. cwd CLAUDE.md 偵測（不存在 → 提示 /init-project-md）
        ↓
8. 輸出載入摘要，等待指令
```

## 八、未來擴展路徑

### 第二階段：知識圖譜（LLM-Wiki）

```
新增：wiki/ + raw/

工作流程：
  你丟資料到 raw/ → 叫 AI「消化這份，更新 wiki/」
  AI 產出變更建議 → 你在 Obsidian 確認 → 寫入 wiki/

wiki/ 結構：
  wiki/index.md              ← AI 維護的目錄
  wiki/log.md                ← 操作紀錄
  wiki/work/<主題>.md        ← 工作知識頁
  wiki/tech/<主題>.md        ← 技術知識頁
```

### 第三階段：維護自動化

```
新增：tools/health-check.sh（純腳本，零 token）

功能：
  - 孤立頁偵測（grep）
  - 斷鏈檢查（grep）
  - 過時頁偵測（find + mtime）
  - 知識庫大小統計

定期（每月）：
  - AI 對照 raw/ 重新驗證 wiki/ 內容
  - 封存 90 天未讀的過時頁
```

### 第四階段：搜尋引擎

```
當 wiki/ 超過 100 頁時：
  加入本地搜尋（BM25 + 向量）
  Claude Code 可透過 MCP 呼叫搜尋
  資料永不離開本機
```

### 情境層（若日後重新設計）

舊版 work/life 情境（含 Secure Mode MCP 白名單）已於這次重構移除。若未來要重新設計，
需要重新決定它跟 `rules/` 三層（global / project 全域 / project path-scoped）如何整合，
而不是照搬舊版的三層優先序（專案 > 情境 > 全域）。

### Schema 版本化（長期）

```
當 SKILL.md 規則演化時：
  在 rules/global.md 加版號
  舊頁面 migrate 到新格式
  避免規則改了但舊知識頁還用舊格式
```
