# ai-workspace — 快速上手指南

---

## 📦 今天先做（安裝前置）

**Step 1：解壓檔案**
```
把 ai-workspace.tar.gz 解壓，得到 ai-workspace/ 資料夾
```

**Step 2：放進 Google Drive**
```
把 ai-workspace/ 整個資料夾放到 Google Drive 同步目錄下
路徑範例：D:\GOOGLE_DRIVE_SYNC\ai-workspace\
```

**Step 3：等 Google Drive 同步完成**
```
確認資料夾內所有檔案都已上傳
```

## 🖥️ 首次安裝（Windows）

**開 PowerShell（系統管理員）**
```powershell
# 進到 ai-workspace 資料夾
cd "D:\GOOGLE_DRIVE_SYNC\ai-workspace"

# 執行安裝腳本
pwsh -ExecutionPolicy Bypass -File setup\setup-windows.ps1
```

**安裝過程會問你：**
- 偵測到舊版 symlink？→ 輸入 `Y` 更新
- 其他詢問 → 看提示選擇

**確認安裝成功**
```
看到以下代表成功：
  C:\Users\你\\.ai-workspace -> D:\GOOGLE_DRIVE_SYNC\ai-workspace  ✓
  C:\Users\你\\.claude\CLAUDE.md -> ...adapters\claude\CLAUDE.md   ✓
  C:\Users\你\\.gemini\GEMINI.md -> ...adapters\gemini\GEMINI.md   ✓
  C:\Users\你\\.codex\AGENTS.md  -> ...adapters\codex\AGENTS.md    ✓
```

## ✍️ 首次使用要填的資料

安裝完後，這個檔案是範本，按需填入真實值：

**全域規則與偏好**
```
檔案：~/.ai-workspace/rules/global.md
填什麼：工作環境、命名慣例、AI 回答偏好（語言、風格等）
```

**MCP 工具（有需要再填）**
```
真實憑證：~/.mcp.env（使用者家目錄，不在 ai-workspace repo 內）
範本參考：~/.ai-workspace/mcp/.env.example
架構設定：~/.ai-workspace/mcp/settings.json
```

## 🗓️ 日常使用流程

```
開 Claude Code
    ↓
context-loader 自動載入 rules/global.md
    ↓
確認專案偵測（Y/n）
    ↓
開始工作

結束時（有重要知識想留存）
    ↓
叫 AI「幫我記住 XXX」→ AI 會判斷寫進 rules 或 mem
```

## ➕ 新增工作專案

```bash
# 1. 複製範本
cp ~/.ai-workspace/projects/_template.md \
   ~/.ai-workspace/projects/新專案名.md

# 2. 填入內容（架構、指令、術語）
# 用編輯器打開填寫

# 3. 在 .gitignore 加入（避免公司 IP 外流）
echo "/projects/新專案名.md" >> ~/.ai-workspace/.gitignore

# 4. 在 context-loader 加入偵測規則
# 編輯 ~/.ai-workspace/skills/global/context-loader/SKILL.md
# 在 Step 3 的表格加一行 Marker Files
```

## 🔄 保持最新

```bash
# SSH / WSL 環境（推薦，含 symlink 校驗）：
bash ~/.ai-workspace/setup/sync.sh

# 上傳自己的變更：
bash ~/.ai-workspace/setup/sync.sh push

# Windows（純 pull，無 symlink 校驗）：
cd D:\GOOGLE_DRIVE_SYNC\ai-workspace
git pull
```
